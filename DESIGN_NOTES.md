# DESIGN_NOTES.md — visual decision log

Append-only. Every UI-touching session adds an entry: what changed, screenshots, what was rejected and why.

## 001 — Founding direction (pre-code)
Direction: "The Astronomer's Workshop" — Prague golem mythology, Orloj, engraved instruments. Midnight ink ground, brass/verdigris/clay/plum accents, Gloock + STIX Two Text + IBM Plex Mono. Signature: LivingPosterior generative chapter headers (real draws as drifting brass curves) + one ceremonial forging animation. Semantic stat palette is load-bearing pedagogy: data=bone, prior=verdigris, posterior=brass, simulation=plum, danger=clay.
Rejected: cream+terracotta editorial serif look (generic AI default, and clashes with dark-workshop identity); neon-on-black tech aesthetic (wrong world — this is brass and candlelight, not cyberpunk); imitating McElreath's lecture-slide style directly (his identity, not ours).
Watchpoints for future sessions: brass on ink fails contrast at small sizes — use --brass-200 below ~16px; don't let "engraved" drift into skeuomorphic textures (no paper grain, no leather); density of instrument panels must never leak into reading pages.

## 002 — Phase 0: tokens, type specimen, home stub (2026-07-04)
Shipped: tokens.css (palette + semantic + stat variables, workshop/daylight via `data-theme`), Tailwind v4 theme mapped to the variables, fonts self-hosted via Fontsource, specimen page at `/dev/specimen`, home as a bench ledger (roman numerals in brass, hairline rows, status marks as tiny engraved circles).

Decisions:
- Semantic layer (`--ground`, `--surface`, `--text-primary`…) sits between palette and components so daylight is a variable swap, not a class swap. Daylight maps `--accent-bright` back to brass-400 — pale brass on bone fails contrast.
- Two hex values live outside the palette table, both documented in place: the `#0B1220` flash guard inline in index.html (first paint before CSS), and daylight `--surface` (#faf6ec), a half-step off bone-100 because ink-800 makes no sense on a bone ground.
- Density curves on the specimen use the exact house treatment from `densityStyle()` so the specimen is contract, not illustration.
Rejected (Chanel rule): "Statistical Rethinking · brms" eyebrow in the top nav — duplicated the hero eyebrow and wrapped to three lines at 390px. The nav is now wordmark + theme toggle only.
Watchpoints: unforged status circles in `--line` are nearly invisible at a glance — deliberate for now (quiet until forged), revisit when real golem SVGs land; daylight mood has no dedicated screenshot pass yet — do one when the first reading page exists.

## 004 — Phase 2: DAG sandbox, explorer, ch 4–6 (2026-07-04)
Shipped: DAG sandbox (radial-free hand-placed nodes, brass = conditioned, live path classification with causal/back-door and open/blocked coloring, OLS bias readout on a truth-anchored number line); Prior Predictive Playground (verdigris prior lines, clay for red-zone violators, live "N of 60 predict impossible people"); Posterior Explorer (brass μ band strictly inside plum predictive band, counterfactual slider with its own plum density); Rosetta code toggle; three golems (Gaussian bell-crown, Multivariable three-node feed, Haunted-DAG ghost-over-collider using plum for the unobserved).
Decisions:
- The stat palette earns its keep in the explorer: μ-uncertainty (brass) vs person-uncertainty (plum) are the same distinction color-coded, and the engine test *enforces* the plum band containing the brass one. Color is load-bearing, not decoration.
- DAG unobserved nodes drawn as dashed `--line` circles that reject clicks — you cannot condition on what you did not measure, and the UI makes that physical.
- Bias readout uses clay for "still fooled" and brass for "honest" so the moral reads at a glance without prose.
- Haunted-DAG golem is the first to use plum in body art (the ghost/confounder), extending the "plum = the unseen simulated thing" logic from charts into illustration.
Rejected: auto-laying-out DAGs with a force sim (nodes drifted unpredictably and broke the hand-tuned compositions); showing numeric coefficients on DAG edges (clutter — the story is which doors are open, not the betas).
Watchpoints: Multivariable golem's three crown-circles crowd on mobile at 130px — check in the Phase 7 mobile pass; explorer PPC plot has ~350 data points drawn as open circles, fine on desktop but worth thinning on mobile; ForgeCTA needed a module-level EMPTY array to keep its zustand selector identity stable (getSnapshot loop) — remember this pattern for any selector returning `?? ## 005 — Phase 3: overfit game, interaction surface, HMC toy, triage (2026-07-04)
Shipped: Overfit-the-polynomial (commit-before-reveal flow: guess chips in plum since the guess is about future data, reveal draws the book's train/test U-plot with brass = in-sample, plum solid = fresh data, plum dashed = LOO); ModelComparison table (mono tabular numerals, brass weight bars, ◆ marks the learner's pick); InteractionSurface (twin panels — conditional line left, slope-vs-moderator "bowtie" right, off-moderator points receded to 18% opacity); HMC toy (density as bone smoke on ink via small-bitmap upscale, trajectories in bone, divergent ones in clay, accepted samples accumulate as brass grains, live q₁ trace); TraceTriage (three chains in brass-400/brass-200/bone-300 — one family, distinguishable; divergence rug in clay); three golems (compass rose with clay monster-points and a brass needle steering between them, manatee in pure clay/bone linework, gyroscope core with a brass marble mid-trajectory).
Screenshots: preview captures during session (bestiary with 8 plinths, HMC funnel desktop + 390px, ch7 game reveal).
Decisions:
- Chain colors in triage stay inside the brass/bone family rather than inventing a categorical palette — chains are all "the posterior being sampled", so they must read as siblings, and the semantic palette survives intact.
- HMC density background switched from per-cell rects (visible moiré grid) to a GRID×GRID ImageData bitmap upscaled with smoothing — reads as smoke, not graph paper. The funnel preset at 390px is the best-looking frame in the app so far.
- Chapter 9's living posterior uses draws from the toy's own engine (2,000 HMC transitions on the ρ=0.9 ridge) — first header where the ornament is literally the chapter's algorithm running.
- Overfit reveal withholds the test/LOO series until the learner commits a degree; the ranked table marks their pick rather than scolding. Predict-before-reveal as interface, not copy.
Rejected: coloring the true data-generating curve verdigris in the overfit game (verdigris means *prior*; the hidden process is not a prior — it renders as dashed bone-300 at 55%); a separate "LOO" hue in the U-plot (dashes on plum carry "estimate of the same plum quantity" without a sixth color); animating triage traces as they draw (the diagnostic skill is reading the still silhouette, and 3×300 path animation stuttered on the low-end profile).
Watchpoints: rugged.csv has semicolons *inside quoted country names* (Micronesia; Iran) — parseSemicolonCsv is now quote-aware, keep using it for every rethinking CSV; HMC canvas ignores devicePixelRatio (slightly soft on retina) — sharpen in the Phase 7 polish pass; ModelComparison weight bars use a fixed 72px max width, revisit if a comparison ever has >8 rows.

## 007 — Phase 7: spaced-review deck (The whetstone) (2026-07-05)
Shipped: the `/review` page — a single-card-at-a-time spaced-repetition drill over authored decks (ch 2–17). One near-square instrument card holds a small-caps kind eyebrow (concept=brass, trap=clay, code=verdigris — the semantic palette carries the card type before you read it), the chapter reference in roman on the right, the prompt in STIX at reading size, an optional mono code block, and a footer that swaps from a single brass "Reveal answer" to four grade buttons. A 1px brass progress rail fills across the session; a due-count badge rides the nav "Review" link in brass.
Decisions:
- The four grade buttons keep "Again" in clay and the other three in brass — failure vs. degrees of success reads from hue, same danger=clay logic as the rest of the app. Each button carries its keyboard number as a faint mono superscript so the shortcut is discoverable without a legend.
- The session is snapshotted once on mount. Grading reschedules a card into the future, and a reactive queue would yank the card out and jitter the "3 of 12" counter — so the queue is frozen and the store updates behind it.
- New (never-seen) cards lead the session, then most-overdue first. You meet a concept before you're drilled on the ones you're forgetting.
- Unlock gate = chapter forged. The deck is a reward for finishing a bench, not a pre-quiz; empty state points you back to the floor to forge one.
Rejected: a categorical per-kind colour set (would have invented hues outside the sacred palette — concept/trap/code map onto brass/clay/verdigris, which already mean posterior/danger/prior and read as "recall / watch out / code"); showing all due cards with no snapshot (counter jitter).
Watchpoints: card header used `justify-between`, which collided the kind label into the chapter roman ("CONCEPTII") once the title wrapped at 375px — now `flex-col` under `sm:`. Never renumber a shipped card id (`s{ch}-{n}`) — it is the SM-2 scheduler key and a rename orphans the learner's schedule for that card.

## 008 — Phase 7 finish: art pass verdict, contrast tokens, the speed of prose (2026-07-05)
Shipped: footer ProgressPorter (export/import as quiet underlined text links — utilities, not actions, so no brass borders); two palette additions forced by measurement, not taste: `--verdigris-300` for links on dark and `--verdigris-600` for links on daylight (verdigris-400 measured 4.06:1 on ink-800 and 3.35:1 on bone — the one hue that failed both grounds), plus `--clay-300` for small clay text on ink. The family reads unchanged; the numbers now clear 4.5:1 everywhere.
Art pass verdict: no changes. All 16 golems hold the engraved line style at plinth size and 375px; the Multivariable crown-circles and Monster Mixer spouts (watchpoints from 004/006) read clearly. The unforged etched-clay state does its job — a bestiary of ghosts until you work.
Performance decisions (these are design decisions):
- Prose is the LCP and never waits for engines. Chapter MDX chunks now contain only prose + KaTeX; every embedded interactive is a lazy component behind `OnApproach` (IntersectionObserver, 600px ahead), holding its bench slot with a bordered "Warming the instrument…" placeholder. Side effect: ch4's in-browser quap fit (~700 ms) now happens as you approach the explorer, not during page load.
- Suspense fallbacks are tall (80–85vh) so a chunk swap never scrolls the footer through the viewport — CLS 0.325 → 0.01.
- Boot prefetch: a direct hit on /chapter/:slug, /bestiary, or /review starts fetching that route's chunks in parallel with React booting. import() dedupes; the router reuses the request.
- STIX 400 + Gloock woff2 are preloaded from index.html by a post-build script — body text is the LCP element, so its font travels with the HTML, not behind the CSS.
Rejected: SSG/prerendering chapter routes (would have cleared 90 by brute force but risks hydration mismatches with canvas + webR components — unnecessary once the embeds went lazy); a Preact swap (same reason, worse risk); per-kind color inventions anywhere.
Watchpoints: `npm run build` now ends with inject-font-preload.mjs — if the fontsource hash pattern changes, the script exits 1 and fails the build loudly; embeds mount on approach, so anything measuring an interactive on mount (tests aside) must scroll it into view first; Lighthouse must be run against a gzip-serving host (vite preview doesn't compress — scores read ~5 points low).
Scores at close (production build, gzipped, simulated mobile): home 98 · bestiary 97 · review 98 · ch2 97 · ch4 94 · ch13 92 · a11y/BP/SEO 100 across the board.
